Final Blaze

Playing as a burning flame we call "Flint", you travel through a factory that's on the brink of breaking down. You are the source the factory depends apon, the energy it runs from. But you were place there against your will and it's time to escape! In Final Blaze you navigate levels jumping over pitfalls, traversing platforms and avoiding certain death from the robots working there as well as the factory itself.
I worked on the enemy AI of this project, handling NPC movement and interaction using a custom state machine in combination with Unity's navigation system.

The Enemy AI

There is 4 components that make up the Enemy System:
* EnemyBase (Enemy/Agent library)
* EnemyProperties (Stats)
* EnemyStateMachine (Transitioning)
* EnemyStates (Logic/Execution)
The enemy is built of an "EnemyBase" script supported by an interface holding its properties. "EnemyBase" owns all the functionality effecting itself and its navigation agent while the state machine handles transitioning logic and executes the active state(s).

State Machine

I built my own statemachine where I decided to make it a bit different from how most normally are done. My machine consists of an active state container, supporting handling multiple states at the same time. This way our enemy can, as an example, be stunned (or overheated as we call it) while getting knocked back at the same time!
Initially the states called to the machine to transition, but I later chose to rework this making the machine switch on states and their different "stages" (enter, update, exit) depending on a toggled flag
